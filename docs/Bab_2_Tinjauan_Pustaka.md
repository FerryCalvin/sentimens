# BAB 2: TINJAUAN PUSTAKA

## 2.1 Analisis Sentimen (Sentiment Analysis)
Analisis Sentimen, atau *Opinion Mining*, adalah cabang dari *Natural Language Processing* (NLP) dan penambangan teks (text mining) yang berfokus pada ekstraksi, klasifikasi, dan penafsiran emosi, opini, atau sentimen masyarakat yang terkandung dalam sebuah teks. Tujuan utama dari analisis sentimen adalah menentukan polaritas emosi teks ke dalam kategori utama, seperti Positif, Negatif, atau Netral. Metode konvensional seperti penggunaan kamus leksikon sentimen telah lama digunakan, namun keterbatasannya dalam memahami konteks dan sarkasme mendorong peralihan menuju pendekatan berbasis *machine learning* dan *deep learning*.

## 2.2 Arsitektur BERT (Bidirectional Encoder Representations from Transformers)
BERT merupakan lompatan revolusioner di bidang NLP yang diperkenalkan oleh Google pada tahun 2018. Sebelum BERT, model bahasa seperti Word2Vec atau GloVe membaca teks secara linear, baik dari kiri ke kanan maupun sebaliknya (satu arah). BERT memperkenalkan konsep pembacaan teks dua arah (Bidirectional) secara simultan menggunakan arsitektur arsitektur *Transformer encoder*. Melalui mekanisme *Self-Attention*, BERT mampu mempertimbangkan konteks kata dari sisi sebelum maupun sesudahnya dalam kalimat. Hal ini menjadikan BERT sangat andal dalam mendeteksi ambiguitas kata, idiom, dan nuansa kalimat kompleks. Dalam prosesnya, model BERT harus diadaptasi pada masalah spesifik seperti klasifikasi teks (Sentiment Analysis) melalui proses yang disebut sebagai *Fine-Tuning*.

## 2.3 Model IndoBERT
Bahasa Indonesia memiliki tantangan tersendiri dalam pemrosesan teks, terutama di media sosial, yaitu tingginya penggunaan bahasa gaul (slang), singkatan, dan penyimpangan struktur tata bahasa baku (SPOK). IndoBERT adalah varian dari model BERT yang dilatih secara khusus (*pre-trained*) pada miliaran kata korpus teks Bahasa Indonesia, yang dikumpulkan dari korpus Wikipedia Indonesia, portal berita, hingga dataset percakapan media sosial. Kumpulan data besar ini membuat IndoBERT memiliki pemahaman *semantic* (makna kalimat) Bahasa Indonesia yang jauh lebih superior dibandingkan BERT versi multilingual, dan telah menjadi standar (*state-of-the-art*) de facto dalam riset NLP berbahasa Indonesia.

## 2.4 Web Scraping dan Ekstraksi Data Asinkron
*Web scraping* merupakan teknik ekstraksi data terstruktur dari halaman antarmuka web secara otomatis menggunakan agen bot atau script komputer. Dalam arsitektur sistem pengolahan opini waktu nyata (real-time), scraping menjadi jembatan pengumpulan dataset untuk dievaluasi oleh mesin NLP. Mengingat sebagian besar portal berita modern dan platform seperti X (Twitter) sangat bergantung pada perenderan Javascript dinamis (Client-Side Rendering), teknik *scraping* konvensional (seperti BeautifulSoup) tidak lagi efisien. Oleh karena itu, penggunaan framework otomatisasi browser (seperti Selenium atau **Playwright**) dengan konsep proses asinkron (*asynchronous I/O*) mutlak diperlukan. Pemisahan kerja antara *network I/O bound* (scraping) dan *CPU bound* (inferensi model) diperlukan agar tidak terjadi kelumpuhan (*bottleneck*) di dalam sistem.

## 2.5 Etika Web Scraping dan Kepatuhan Kebijakan
Praktik pengumpulan opini publik (*scraping*) sering bersinggungan dengan ranah etika teknologi informasi dan legalitas (kepatuhan terhadap *Terms of Service* suatu layanan). Dalam pengembangannya, sebuah sistem scraper diwajibkan untuk:
1. Mematuhi batasan rasio request (rate limiting) agar tidak membebani server target layaknya serangan *Denial of Service* (DoS).
2. Mengekstraksi hanya data yang bersifat publik (Open Source Intelligence/OSINT), tanpa menerobos dinding privasi, data sensitif PII (Personally Identifiable Information), atau sistem berbayar.
3. Mensimulasikan `User-Agent` manusiawi (realistic browser fingerprints) secara jujur ketika tidak menggunakan API resmi, sebagai bentuk perlindungan identitas bot dan transparansi riset. Pada "Sentiments", pengambilan data dilakukan pada ranah publik semata-mata demi validasi pengolahan bahasa alami, tanpa re-distribusi data mentah.

## 2.6 State-of-the-Art (Penelitian Terkait)
Dalam beberapa tahun terakhir, penelitian NLP Bahasa Indonesia didominasi oleh perbandingan arsitektur *Machine Learning* klasik dengan *Deep Learning*.
- Penelitian oleh Koto et al. (2020) mengenai pengenalan **IndoLEM** dan **IndoBERT** membuktikan bahwa pre-training spesifik bahasa meningkatkan poin akurasi secara drastis dalam metrik F1-Score untuk *Sentiment Analysis* (mencapai batas >85% pada dataset IndoNLU).
- Kajian implementasi infrastruktur web untuk Machine Learning merekomendasikan pemisahan (decoupling) orkestrasi web API; seperti mendelegasikan komputasi neural network pada web server ringan namun handal (*FastAPI* atau *Flask*) dengan pemrosesan *queue* di latar belakang untuk menekan isu *timeout* pada arsitektur monolitik. "Sentiments" mereplikasi *best-practice* ini dengan Microservices tersimulasi (Flask mengatur presentasi dan routing, API terpisah mengeksekusi ekstraksi web).
